## 二、需现场准备（无法通用打包）

### 1. 编译工具链（Redis / Nginx 源码编译用）★最关键

纯无网络下 `yum/apt install gcc` 会失败。三选一：

- **A. 系统安装 ISO 配本地 yum 源**（推荐）：挂载装机 ISO，配 file:// 本地源，即可离线 `yum install gcc gcc-c++ make`。Nginx 还需 `pcre-devel zlib-devel openssl-devel`。这是私有化现场最标准做法。
- **B. 现场同款系统预编译一次**：在一台同发行版同架构的机器上编译好 Redis/Nginx，把二进制打包，后续机器直接分发（本工具支持放预编译二进制则跳过编译，见三）。
- **C. 用系统自带包**：麒麟/欧拉的系统源常自带 redis、nginx 的 rpm，若现场有本地源可直接 `yum install redis nginx`（版本以系统源为准）。

> 为什么不预编译好放离线包：ubuntu(glibc 2.35) 编的二进制在 CentOS 7(glibc 2.17) 无法运行；反之亦然。必须目标同环境编译。

### 2. 达梦建表 SQL 的执行

Nacos / XXL-JOB 用达梦时，需先在达梦库建 schema 并执行建表 SQL（离线包已带 SQL）。执行需**达梦客户端 disql**（达梦安装 ISO 内自带 `dmdbms/bin/disql`）。本工具装完会提示 SQL 路径，DBA 用 disql 执行。MySQL 模式同理需 mysql 客户端。

### 3.（可选）Kafka

手册 14 组件不含 Kafka，但你在 `F:\阿克苏镜像准备\` 已放 Kafka 部署文档。如需纳入自动化，告知版本/部署方式（容器 or 二进制），另做安装器。

## 三、工具已做的无网络适配

- **Nacos 解压**：不依赖 `unzip`，无 unzip 时自动用 JDK 自带的 `jar` 解压。
- **Docker**：静态二进制，完全不碰 yum/apt；老内核（CentOS 7）自动改用 20.10 兼容版。
- **Redis 编译**：先检测 gcc/make，已有直接编译；缺失且无法安装时明确报错指向本清单，不再无意义地卡 yum 超时。
- **架构自适应**：所有组件按 `uname -m` 自动选 x86_64 / arm64 包。
- **容器组件**：`docker load` 本地镜像 tar，不拉网络。

## 四、一句话总结

**能打包的都打包了（含双架构镜像）。现场唯一必须自备的是"编译工具链"——通过挂载系统安装 ISO 配本地 yum 源解决，这是私有化交付的标准动作。** 达梦客户端随达梦 ISO 自带。
